MongoDB在ubuntu server下的部署

把实验室里的一台IBM 3250翻了出来用来跑MongoDB

首先

下载MongoDB

wget http://fastdl.mongodb.org/linux/mongodb-linux-i686-2.2.3.tgz

 

解压安装

$ tar -zxvf mongodb-linux-i686-2.2.3.tgz
$ sudo mv mongodb-linux-i686-2.2.3.tgz /usr/local/mongodb

 

配置用户及目录

$ sudo useradd -r mongodb

#日志文件路径
$ sudo mkdir /var/log/mongodb

数据文件路径

$ sudo mkdir /var/lib/mongodb
$ sudo chown mongodb /var/lib/mongodb
 

使用配置文件

配置文件地址（https://github.com/adamwen829/mongodb-conf）

如果在前面你更改了log或者data的地址 那么配置文件也需要做相应的更改

 

$ git clone git://github.com/adamwen829/mongodb-conf.git
$ cd mongodb-conf/etc/init.d/
$ sudo cp mongodb /etc/init.d/
$ sudo chmod +x /etc/init.d/mongodb
$ cd ..
$ sudo cp mongodb.conf /etc/

 

启动关闭monogodb

#启动mongodb
$ sudo /etc/init.d/mongodb start

#关闭mongodb
$ sudo /etc/init.d/mongodb stop
 

看到如下信息，说明db安装成功

  • Starting database mongodb [ OK ]

 

开机启动

$ sudo update-rc.d mongodb defaults

我该如何写标题来形容操蛋的学院

放假回家一周了，项目还没有忙完。那套考试系统有点不知道该怎么继续写下去，有点像过度优化，但是想了想决定还是先写，想多了不是好事，在现有的基础上加上题库选题模块，应该没有问题。

期末考完试在学校里又呆了一周，为了这个考试系统，总结一下：

1.Trello是个好东西

2.接口是先协商好很重要

3.版本控制很重要

4.我需要收敛我的脾气

 

有时候真心给学院跪了，感觉心灰意冷。操蛋的课程，再加上基本上什么都不管的学院，现在想想我很后悔的是我并没有把翘掉的课的时间完全高效的利用。假期在哪里给学院做东西，学院给的支持竟然只有一张很破的双人床，没有提供宿舍住宿，没有提供其他的，就一张双人床，比学生宿舍里还要破烂的双人床，我真是呵呵了。现在只是觉得学院就是拿学生当做赚钱的工具，再然后呢？我不知道。

幸好有江认识个副校长，我们才能得以住进学生宿舍。

我尊敬学院里的老师，因为我觉得他们最起码是在教我们，而且学院里还是有几个特别好的老师，比如李老师，比如卢毅鸿老师，比如巴叔，都是在对学生认真负责，但是关于学院，呵呵了。听说过于佳老师带ACM两个月学院给发了500的事情，也听说过学院的5W的事情，让人心寒。

给张床让我做东西我忍了，没办法，我还想毕业。东西的质量我会尽力保证，我觉得出自自己之手的东西还是要对自己负责的，至于学院所谓的补贴，我只剩两点希望：

1.别让李老师自己垫上

2.正如和其他几个人开玩笑所说的，别比我们留校期间自己的花费少太多就行。

关于学院，我实在难抱所谓的感恩之心。对于我的小学，初中，高中，我都十分喜欢并尊重，但是对于现在的学院，让我明白了为什么学院里没有所谓的什么毕业之后回报学院之类的。

也许当我工作之后，我愿意为创新实验室持续捐购图书或者提供别的支持，但是对于学院，我只愿以后提到它的时候，我的成熟能够让我不再显露气愤。

2012年度小结(一)

一年快要过去了，回想一下这一年，从年初到现在。

刚刚过年的时候也是分手的时候，人生迎来一个最大的低潮时期。整个人都是处于浑浑噩噩的状态。其实早就有预兆，只是自己不肯承认罢了。点破了果然还是伤心。一个寒假瘦了10斤。依然心不死，唉，想想就感觉自己很贱。毕竟是初恋始终割舍不下，对这份感情，写了很多，想了很多。

寒假的时候逼着自己做了几道题，算是为软件大赛做准备。回来之后就开始进入了类似有点集训的状态。刷题，做题，顺利的进入决赛，在决赛赛场是把一个题边吃汉堡边调试了一个半小时，最后拿了个全国二等奖，可惜没拿一等奖，算是一个遗憾吧。

春季学期期末的时候刘骁从英国回来，我依然还记得那个晚上我和刘骁在奥帆中心看着那深邃幽黑的海水在聊天，关于未来，关于社会，关于我们。有时候也感觉想的太多不好，总会让自己的生活感到沉重，有点希望像大伟二挂那样可以享受生活，到处寻找各种好玩的，但是每个人都有自己的路要走。

假期的时候，基本上一个假期都在学校里。写代码，看书，和同学扯淡。就是这样，而且感觉从假期开始，整个人开始浮躁，但也是从假期开始，好像心情不再是一直压抑的状态，假期找了一份实习，可惜总是感觉很清闲的样子，始终找不到感觉，到了开学就辞掉了实习，然后在假期 课程同步助手 开始上线，上线第二天就看到了 课程盒子 这个软件， 让我更加深刻的认识了一句话， 你认为是一个创新的idea有课程早已经被别人想过千万遍。当然 我也没打算商业化， 就是为了能让大家更加方便的使用， 哪个方便用哪个，所以我也没在人人上对课程盒子发表任何评论。

9月，SFD青岛站顺利举行。因为准备的有点晚，有些赞助没有拿到。此次OSQDU作为主办方，OSQDU目前在青大的就只有我一个人。所以忙的焦头烂额，因此也特别感谢实验室那些所有帮助过我的兄弟。这次SFD还算是成功吧，感谢所有的演讲嘉宾，感谢实验室帮助过我的哥们，也感谢所有的到场观众。

11月，空教室查询系统上线。Alex和我说英文名有点不太对，我还以为是哪个单词拼错了，后来是发现我把教室查询写成了课程查询。上线之后还算是平稳。不过最近让我给停了,等到考试之后新版准备上线.

 

12月,hackathon.其实过程还是不错的,但是最后的评选让人觉得伤心,各大公司派来的一些非技术相关人员,然后评调用自家平台的东西给奖.但是我们24小时的开发还是十分给力的,计划假期继续维护那个项目.突然觉得hackathon应该单独写一篇来回忆.回来没有多久就6级考试,然后到现在开始准备期末考试.

从GURUDigger的一个bug说起

申请gurudigger的邀请码支持没得到系统回应，就直接发邮件要了一个。

注册完完了完 然后发现了一个比较严重的bug

GURUDIGGER采用了 http://gurudigger.com/users/username 来指向个人主页，就是直接将username放入url中作为标定的用户。

但是GURUDIGGER在检查用户名是否重复的时候没有忽略大小写，这有什么问题呢？问题就是在我注册adam这个用户之前，系统里已经存在了一个用户名为Adam的用户，但是因为系统不做大小写忽略，我得以成功注册。当注册之后，查看个人主页，我们知道url是不区分大小写的，而Adam注册之前，所以当我想访问我自己的个人主页，即http://gurudigger.com/users/adam时，总是显示的是Adam的信息。

 

如果要将数据库的中的数据放入url中，注意大小写问题，因为url是忽略大小写的，所以你的数据库逻辑在处理这一数据时，应该也是忽略大小写的，如果不忽略大小写， （写完后又查了查，域名是不区分大小写的，至于后面的url，由系统决定，因为windows本身的文件系统不区分大小写，所以win服务器不区分大小写，而linux/unix服务器区分大小写）那么应该可以用哈希的方法标定出一个唯一的不重复id来识别用户的身份。

 

好的系统不代表没有bug的系统， bug无处不在。。。。

一个软件， 重要的到底是什么

最近和同学开发一个网站，也算是cms吧 人一多遇到的全是坎坷。到底是先分工再写，还是如何。接口自始至终都没有完整的定义过，编写边改 哪怕是写的时候 也没有接口说明作为参照。在我看来能开发出这套系统确实令人惊叹。

后台写完了等前端，等了一段时间等到前端写完了，发现双方留的接口完全对不上，然后有一方选择改动。陷入了零和博弈，简单的说，前端和后台分为两组，必然有一方要改动，总有一方可以快乐的不改，但是这种快乐必然是建立在另一组的痛苦之上。写的无比艰辛，甚至让我怀疑我是否真的会编程。明明知道自己能力有限，做常常纠结与一个类怎么能写的更好，在未来可能会更具有扩展性，结果一个类一改再改。

在写这套系统的时候，算法和数据结构，反而成为了次要的东西，软件一开始的接口的说明， 如何设计这套系统耦合， 单独的模块怎么算测试通过，两个模块怎么耦合， 更多的是属于软件工程和设计模式的东西。

也算是边学边做，有过django的基础，边学pyramid边用， 顺便又熟悉了不少应用层上的东西。算是了解了登录系统的原理，然后又加了一层来让android不持有cookie也能通过同一套系统的验证。然后又设计了一套通知系统，sns的那种，不过估计只是勉强能用，一旦遇到海量数据就必崩无疑。

 

一套系统是不是开始的设计主要是看架构，然后才考虑性能呢。总觉得一开始过分的关注细枝，拖慢了开发进程。

 

也算是第一次有了多人使用git的经验，要学习的还有 很多
